
# Юнит‑тесты OfferWallService

Эти тесты проверяют слой приложения (use‑cases) без БД и фреймворка, используя мок‑репозиторий, совместимый с портом `OfferWallRepository (Protocol)`.

## Что покрыто
- `list_offerwalls()` — возврат элементов и фильтрация по имени.
- `get_offerwall()` — успешный результат и 404 (`NotFoundException`) при отсутствии.
- `get_offer_names()` — возвращает непустой список строк из источника имён.

## Запуск
.\.venv\Scripts\pytest.exe -q tests\unit\test_offerwall_service.py

## Примечания
- Мок‑репозиторий реализует контракт порта и работает на данных в памяти.
- Тесты вызывают асинхронные методы через `asyncio.run(...)`, поэтому не требуют `pytest-asyncio`.
- Эти тесты устойчивы к изменениям инфраструктуры и соответствуют принципам Чистой Архитектуры: слой приложения тестируется изолированно от драйверов (ORM/HTTP).
